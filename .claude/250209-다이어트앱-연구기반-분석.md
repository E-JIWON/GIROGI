# GIROGI 다이어트 앱 - 연구 기반 전략 분석 보고서

**분석 일자**: 2026-02-09
**분석 대상**: GIROGI (기로기) 다이어트 앱 (Flutter MVP, Next.js 전환 중)
**분석 목적**: 5가지 연구 기반 다이어트 전략이 앱에 제대로 적용되었는지 검증

---

## 🎯 분석 개요

### 사용자가 제시한 5가지 연구 전략

1. **Episodic Future Thinking (EFT)** - 미래 상상 훈련
2. **Temptation Bundling** - 유혹 묶기
3. **Implementation Intention** - 실행 의도
4. **Tiny Habits** - 아주 작게 시작하기
5. **Environment Design** - 환경 설계

### Spec 문서에 추가로 정의된 원리

6. **Self-Compassion** - 자기 연민 (실패 후 복귀)
7. **Slow Eating** - 천천히 씹기

**강점**:
- 5가지 핵심 전략 모두 앱 기능으로 매핑됨

## 🔬 2단계: 연구 의도대로 구현되었는가 심층 분석

### ✅ 1. Episodic Future Thinking (EFT)

**연구 의도**:
- 유혹 상황에서 "건강해진 미래의 나"를 생생하게 상상
- 칼로리 섭취 약 99kcal 감소, BMI 감소 효과

**앱 구현 (`FutureSelfCard`)**:
```dart
// 목표 이미지 표시
goalImageUrl: null  // ⚠️ 기본값 null, 사용자가 업로드해야 함

// 동기부여 메시지 (하드코딩)
'• 지금 먹으면 후회할 것 같나요?'
'• 목표 달성이 늦어질 수 있습니다'
'• 10분만 기다리면 유혹이 사라집니다'
'• 미래의 내가 고마워할 선택을 하세요'
```

**✅ 잘된 점**:
- 목표 이미지, 체중 차이, D-day 카운트다운 모두 구현
- 유혹 극복 화면에서 적시에 표시 (타이밍 적절)
- 시각적 강조 (카드 디자인, 아이콘)

**⚠️ 개선 필요**:
1. **목표 이미지가 기본값 null**: 사용자가 설정하지 않으면 플레이스홀더만 표시
   - **문제점**: EFT 효과는 "생생한 이미지"가 핵심인데, 이미지 없으면 효과 반감
   - **제안**: 온보딩에서 목표 이미지 설정을 필수로 만들거나, 기본 이미지 제공

2. **동기부여 메시지가 정적**: 항상 같은 4줄 표시
   - **문제점**: 반복 노출 시 익숙해져서 효과 감소
   - **제안**: 10가지 메시지를 랜덤으로 표시하거나, 시간대별 맞춤 메시지

3. **유혹 트리거 없음**: 사용자가 직접 "유혹 극복" 화면에 들어와야 함
   - **문제점**: 유혹이 느껴질 때 능동적으로 앱을 열기 어려움
   - **제안**: 외식/야식 시간대(저녁 8시 이후)에 푸시 알림 자동 발송

**종합 평가**: ⭐⭐⭐⭐☆ (구현은 잘했으나 트리거 부족)

### ✅ 2. Temptation Bundling (유혹 묶기)

**연구 의도**:
- 좋은 행동 + 하고 싶은 것 묶기
- 운동 참여율 10-14% 증가

**앱 구현 (`RewardStatusCard`, `app_constants.dart`)**:
```dart
// 과자박스: 3일 연속 성공 → 1개 획득
static const int daysForSnackBox = 3;

// 치팅데이: 7일 연속 성공 → 과자박스 사용 가능
static const int daysForCheatDay = 7;

// 자동 계산
int get daysUntilCheatDay {
  final remaining = AppConstants.daysForCheatDay -
    (consecutiveDietDays % AppConstants.daysForCheatDay);
  return remaining == AppConstants.daysForCheatDay ? 0 : remaining;
}
```

**✅ 잘된 점**:
1. **명확한 보상 규칙**: 3일, 7일 상수로 정의되어 있음
2. **시각적 피드백**: 과자박스 개수, 치팅데이 카운트다운 표시
3. **즉각적 보상 + 지연 보상 조합**: 단기(3일)/장기(7일) 동기 모두 충족
4. **상수 중앙 관리**: `app_constants.dart`에서 쉽게 수정 가능

**⚠️ 개선 필요**:
1. **보상 사용 메커니즘 미구현**: 과자박스를 "사용"하는 화면이 없음
   - **문제점**: 보상을 받아도 실제로 쓸 수 없으면 동기부여 약함
   - **제안**: "과자박스 사용하기" 버튼 → 치팅데이에 사용 가능, 사용 시 기록 남기기

2. **보상 규칙 설명 부족**: 처음 보는 사람은 "과자박스가 뭐지?" 의아할 수 있음
   - **제안**: 홈 화면에 "보상 규칙 보기" 버튼 추가, 온보딩에서 설명

**종합 평가**: ⭐⭐⭐⭐⭐ (연구 의도 완벽 반영, 사용 메커니즘만 추가하면 완성)

### ⚠️ 3. Implementation Intention (실행 의도)

**연구 의도**:
- "언제 + 어디서 + 무엇을" 형식으로 구체화
- 목표 달성률 2-3배 증가

**앱 구현 (`ChecklistScreen`)**:
```dart
// 아침 체크리스트
{'title': '물 한 잔 마시기', 'isChecked': true},
{'title': '체중 측정하기', 'isChecked': true},

// 점심 체크리스트
{'title': '식사 전 물 한 잔', 'isChecked': true},
{'title': '천천히 먹기 (20분 이상)', 'isChecked': false},
```

**❌ 문제점**:
1. **"언제+어디서+무엇을" 형식이 아님**: 단순 체크리스트
   - 예: "물 한 잔 마시기" → "아침 10시에 사무실에서 물 500ml 마시기"로 바꿔야 함
   - 현재는 "무엇을"만 있고 "언제", "어디서"가 모호

2. **시간대 구분만으로는 부족**: "아침", "점심", "저녁"은 너무 광범위
   - Peter Gollwitzer 연구: "When I see X, I will do Y" 패턴이 핵심
   - 현재: "아침에 물 마시기" (막연함)
   - 이상적: "알람이 울리면(X) 물을 마신다(Y)"

3. **사용자 맞춤 설정 불가**: 하드코딩된 체크리스트
   - 바쁜 직장인은 "10:30 물 500ml"가 맞지만, 자영업자는 다를 수 있음

**✅ 잘된 점**:
- 시간대별 그룹화는 Implementation Intention의 기초는 충족
- "30회 이상 씹기"처럼 구체적 숫자 포함

**종합 평가**: ⭐⭐⭐☆☆ (시간대 그룹화만으로는 불충분, "When-Where-What" 형식 필요)

**제안**:
1. 체크리스트 항목 형식 변경:
   ```
   기존: "물 한 잔 마시기"
   개선: "🕐 10:30에 💼 사무실에서 💧 물 500ml 마시기"
   ```

2. 체크리스트 커스터마이징 기능 추가:
   - 시간, 장소, 행동을 사용자가 직접 설정
   - 템플릿 제공 (직장인용, 재택근무용, 자영업자용)

### ✅ 4. Tiny Habits (아주 작게 시작하기)

**연구 의도**:
- 뇌가 저항하지 않을 만큼 작은 단위로 습관 형성
- 예: "매일 운동" ❌ → "매일 스쿼트 1개" ✅

**앱 구현 (`app_constants.dart`, `MissionCard`)**:
```dart
// 미션 개수: 3개 (적은 개수)
static const int totalCoreMissions = 3;

// 성공 조건: 3개 중 2개만 달성하면 OK (완벽주의 회피)
static const int minCoreMissionsForSuccess = 2;
```

**✅ 잘된 점**:
1. **"3개 중 2개" 규칙**: 완벽주의를 피하고 심리적 부담 감소
   - BJ Fogg 연구 핵심: "작고 쉽게 → 저항 없이"
   - 5개가 아닌 3개, 3개 모두가 아닌 2개 → 매우 과학적

2. **구체적이고 작은 미션**:
   - Spec 문서: "퇴근 전 바나나 먹기", "스쿼트 1개", "작은 그릇 사용"
   - 모두 5분 이내 완료 가능한 작은 행동

3. **상수로 관리**: 미션 개수를 바꾸고 싶으면 `totalCoreMissions`만 수정하면 됨

**⚠️ 개선 필요**:
1. **미션이 하드코딩됨**: 모든 사용자가 같은 3개 미션
   - 문제점: "바나나 싫어하는 사람", "헬스장 안 다니는 사람"은 억지로 해야 함
   - 제안: 미션 풀(Pool)에서 사용자가 3개 선택하게 하기

2. **"작게 시작"의 점진적 증가 없음**: 계속 스쿼트 1개만 하면 효과 제한적
   - 제안: 1주차 스쿼트 1개 → 2주차 스쿼트 3개 → 3주차 스쿼트 5개 (점진적 증가)

**종합 평가**: ⭐⭐⭐⭐⭐ (연구 의도 완벽 반영, 점진적 증가만 추가하면 더 좋음)

### ⚠️ 5. Environment Design (환경 설계)

**연구 의도**:
- 의지력 대신 환경을 바꿔 행동 유도
- 예: 배달앱 삭제, 작은 그릇 사용, 과자 안 보이는 곳에 두기

**앱 구현**:
- ✅ "작은 그릇 사용" 체크리스트 항목 (간접적 유도)
- ✅ 외식 횟수 경고 (`weeklyDiningOutWarningThreshold = 3`)
- ❌ **배달앱 차단, 알림 설정 등 직접적 환경 설계 기능 없음**

**❌ 문제점**:
1. **앱 내에서 환경을 바꿀 수 없음**:
   - 배달앱을 앱 내에서 차단할 수 없음 (iOS/Android 제한)
   - 하지만 "배달앱 삭제 리마인더"는 가능

2. **푸시 알림 미구현**:
   - Spec에는 있으나 구현 안 됨 (Phase 12 이후)
   - 예: "저녁 8시에 '8시 전 식사 완료했나요?' 알림"

3. **리마인더 시스템 없음**:
   - "10:30 물 500ml" 체크리스트 항목은 있지만, 10:30에 알림이 오지 않음

**✅ 잘된 점**:
- 외식 경고 (3회 초과 시) → 간접적으로 외식 줄이게 유도
- "작은 그릇 사용" 같은 구체적 환경 설계 항목 포함

**종합 평가**: ⭐⭐☆☆☆ (앱 내에서 직접 환경을 바꿀 수 없어서 약함)

**제안**:
1. **푸시 알림 우선순위 높임** (Phase 12에서 가장 먼저 구현)
   - 10:30 물 리마인더
   - 저녁 8시 "지금 먹으면 늦어요" 경고
   - 23:55 "Streak 위험!" 긴급 알림

2. **환경 설계 체크리스트 추가**:
   - 온보딩에서: "배달앱을 삭제하셨나요?" (Yes/No)
   - "과자를 눈에 안 보이는 곳에 두셨나요?"
   - → 체크하면 앱이 주기적으로 리마인드

3. **앱 사용 시간 제한 연동**:
   - iOS Screen Time, Android Digital Wellbeing API 활용
   - "배달앱 사용 제한 설정하기" 버튼 → OS 설정으로 이동

### ✅ 6. Self-Compassion (자기 연민)

**연구 의도**:
- 실패를 성장의 기회로 전환, 자기비난 최소화
- 실패 후 복귀율 증가, 다이어트 지속 의지 증가 (British Journal of Health Psychology 2021)

**앱 구현 (`SelfCompassionCard`, `FailureReportDialog`)**:
```dart
// 자기 연민 메시지
'한 번의 실수로 모든 것이 끝나는 건 아닙니다'
'전체 성공률: 84%' // 실패 1번 ≠ 전부 실패

// 실패 리포트: 3단계 질문
'어떤 상황이었나요?'
'왜 실패했나요?'
'다음엔 어떻게 할까요?'
```

**✅ 잘된 점**:
1. **자기비난 방지**: "실패!" 대신 "괜찮아요" 메시지
2. **전체 성공률 표시**: 실패 1번을 전체 맥락에서 봄 (긍정적 프레이밍)
3. **즉각 복귀 버튼**: "다시 시작하기" 버튼으로 즉시 행동 유도
4. **실패 리포트**: 원인 분석 → 학습 기회로 전환
5. **커뮤니티 공유**: 실패를 공유하면 응원 받음 (사회적 지지)

**⚠️ 개선 필요**:
1. **"허가 효과(Licensing Effect)" 경고 문구가 너무 약함**:
   - Spec에는 "내일부터 하자는 허가 효과" 경고가 있으나 코드에서는 미확인
   - 제안: "⚠️ '내일부터'는 금물! 지금 바로 다시 시작하세요"

2. **실패 리포트가 선택 사항**:
   - 문제점: 사용자가 건너뛰면 학습 기회 상실
   - 제안: Streak 리셋 시 실패 리포트 작성을 권장 (팝업)

**종합 평가**: ⭐⭐⭐⭐☆ (연구 의도 잘 반영, 허가 효과 경고만 강화하면 완성)

### ✅ 7. Slow Eating (천천히 씹기)

**연구 의도**:
- 20-30번 씹기로 포만감 증가, 식이 유발 열 생성 증가
- Scientific Reports (2021), Nutrients (2025)

**앱 구현**:
```dart
// 체크리스트
{'title': '천천히 먹기 (20분 이상)', 'isChecked': false},
{'title': '30회 이상 씹기', 'isChecked': false},

// 상수
static const int targetChewCount = 30;
```

**✅ 잘된 점**:
- 구체적 숫자 제시 (30회)
- 시간 기준도 추가 (20분)

**⚠️ 개선 필요**:
1. **씹는 횟수를 어떻게 세는가?**:
   - 문제점: 사용자가 직접 세기 어려움
